# 群通知功能 - 前端对接文档

## 一、概述

群通知是一类**系统通知型**消息，用于在群通知/公告页面显示。

### 协议概览

| 协议号 | 协议类型 | 触发场景 | 接收者 |
|--------|---------|---------|--------|
| **MT52** | 入群申请通知 | 用户申请加入需审核的群 | 管理员/群主 |
| **MT53** | 入群审核结果通知 | 管理员审核入群申请 | 申请人 |
| **MT54** | 群管理操作通知 | 管理员进行权限变更等操作 | 被操作人/管理员 |

### 与普通群聊消息的区别

| 类型 | 显示位置 | 触发者 | 接收者 |
|------|---------|--------|--------|
| 普通群聊消息 | 聊天消息列表 | 任意群成员 | 所有群成员 |
| **群通知** | **群通知页面** | **管理员/群主/申请人** | **相关人员** |

## 二、MT52 - 入群申请通知

### 协议号

```javascript
MT52 = 52  // 入群申请通知
```

### 触发场景

用户申请加入需要审核的群时，向管理员/群主发送通知。

### 协议体结构

```typescript
interface JoinRequestNotification {
    /** 通知类型固定为 "join_request" */
    type: "join_request";
    
    /** 申请人UID */
    applicant_uid: string;
    
    /** 申请人昵称 */
    applicant_nickname: string;
    
    /** 申请人头像URL */
    applicant_avatar: string;
    
    /** 申请说明 */
    request_desc: string;
    
    /** 群ID */
    gid: string;
    
    /** 申请记录ID（用于审核时传入） */
    request_id: number;
    
    /** 通知展示内容 */
    notification_content: string;
    
    /** 父指纹码 */
    parent_fp: string;
}
```

### 通知内容模板

`"张三"申请加入群聊`

### 接收者

管理员（role >= 1）和群主

### 示例通知内容

```
"张三"申请加入群聊
申请理由：我是李四的朋友
```

## 三、MT53 - 入群审核结果通知

### 协议号

```javascript
MT53 = 53  // 入群审核结果通知
```

### 触发场景

管理员审核用户的入群申请后，向申请人发送结果通知。

### 协议体结构

```typescript
interface JoinReviewResultNotification {
    /** 通知类型固定为 "join_review_result" */
    type: "join_review_result";
    
    /** 审核人UID */
    reviewer_uid: string;
    
    /** 审核人昵称 */
    reviewer_nickname: string;
    
    /** 群ID */
    gid: string;
    
    /** 群名 */
    gname: string;
    
    /** 是否通过审核 */
    approved: boolean;
    
    /** 拒绝原因（仅拒绝时有值） */
    reject_reason: string;
    
    /** 通知展示内容 */
    notification_content: string;
    
    /** 父指纹码 */
    parent_fp: string;
}
```

### 通知内容模板

| approved | notification_content |
|----------|---------------------|
| `true` | `"你的入群申请已通过审核"` |
| `false` | `"你的入群申请已被拒绝"` 或 `"你的入群申请已被拒绝：XXX"` |

### 接收者

申请人（发起入群申请的用户）

### 实时推送

**通过后会自动通过 WebSocket 实时推送 MT53 通知给用户**，无需前端轮询。

### 通过后的实时通知

当管理员审核通过时，申请人会**实时收到 WebSocket 推送**：

```javascript
// WebSocket 接收 MT53 通知
function onMessage(data) {
    if (data.typeu === 53) {
        const notification = JSON.parse(data.msg);
        handleJoinReviewResult(notification);
    }
}

function handleJoinReviewResult(notification) {
    if (notification.approved) {
        // 申请通过！显示通知
        showNotification({
            icon: "🎉",
            title: "入群申请已通过",
            message: `你已成功加入「${notification.gname}」`,
            action: () => {
                // 跳转到群聊页面
                navigateToGroupChat(notification.gid);
            }
        });
    } else {
        // 申请被拒绝
        showNotification({
            icon: "❌",
            title: "入群申请被拒绝",
            message: notification.reject_reason 
                ? `原因：${notification.reject_reason}` 
                : "管理员拒绝了你的入群申请"
        });
    }
}
```

## 四、MT54 - 群管理操作通知

### 协议号

```javascript
MT54 = 54  // 群管理操作通知
```

### 协议体结构 (CmdBody4GroupAdminNotification)

```typescript
interface GroupAdminNotification {
    /** 通知类型 */
    notifyType: string;
    
    /** 操作者UID */
    operatorUid: string;
    
    /** 操作者昵称 */
    operatorNickname: string;
    
    /** 目标用户UID（被操作人，可为null） */
    targetUid: string | null;
    
    /** 目标用户昵称 */
    targetNickname: string | null;
    
    /** 群ID */
    gid: string;
    
    /** 群名 */
    gname: string;
    
    /** 通知内容 */
    content: string;
    
    /** 额外数据（JSON），如禁言时长等 */
    extraData: string | null;
    
    /** 创建时间戳 */
    createTime: number;
    
    /** 通知记录ID（用于已读标记等） */
    notificationId: number;
}
```

## 五、通知类型

### 通知类型枚举（MT54）

| notifyType | 场景 | 通知范围 | 说明 |
|------------|------|---------|------|
| `admin_set` | 设置管理员 | 被设置者 + 其他管理员 | ✅ 实时推送 |
| `admin_remove` | 取消管理员 | 被取消者 + 其他管理员 | ✅ 实时推送 |
| `transfer_owner` | 转让群主 | **仅新群主** | ✅ 实时推送 |
| `dismiss_group` | 解散群聊 | **所有群成员** | ✅ 实时推送 |

### 特殊通知说明

#### 1. 转让群主 (transfer_owner)

| 接收者 | 通知内容 | 说明 |
|--------|---------|------|
| **新群主** | `"你已成为群主"` | 仅通知新群主一人 |
| 原群主 | ❌ 不通知 | 转让发起人不需要通知自己 |

```javascript
// 转让群主通知示例
{
    "notifyType": "transfer_owner",
    "operatorUid": "原群主UID",
    "operatorNickname": "张三",
    "targetUid": "新群主UID",      // 被转让者
    "targetNickname": "李四",
    "gid": "群ID",
    "gname": "群名称",
    "content": "你已成为群主",
    "extraData": null
}
```

#### 2. 解散群聊 (dismiss_group)

| 接收者 | 通知内容 | 说明 |
|--------|---------|------|
| **所有群成员** | `"该群已解散"` | 包括原群主在内所有人 |

```javascript
// 解散群通知示例
{
    "notifyType": "dismiss_group",
    "operatorUid": "原群主UID",
    "operatorNickname": "张三",
    "targetUid": null,
    "targetNickname": null,
    "gid": "群ID",
    "gname": "群名称",
    "content": "该群已解散",
    "extraData": null
}
```

## 六、接口对接

### 1. 接收 WebSocket 推送

MT52、MT53、MT54 协议消息通过 WebSocket 实时推送，前端需要解析对应 `typeu` 的消息。

```javascript
// WebSocket 消息解析示例
function onMessage(data) {
    switch (data.typeu) {
        case 52:
            // 入群申请通知（发给管理员）
            handleJoinRequestNotification(data.msg);
            break;
        case 53:
            // 入群审核结果通知（发给申请人）
            handleJoinReviewResultNotification(data.msg);
            break;
        case 54:
            // 群管理操作通知
            handleGroupAdminNotification(data.msg);
            break;
    }
}

function handleJoinRequestNotification(msg) {
    const notification = JSON.parse(msg);
    // notification.type === "join_request"
    // 显示：申请人 XXX 申请加入群聊
}

function handleJoinReviewResultNotification(msg) {
    const notification = JSON.parse(msg);
    // notification.type === "join_review_result"
    // notification.approved === true/false
    // 显示：你的入群申请已通过/被拒绝
}

function handleGroupAdminNotification(msg) {
    const notification = JSON.parse(msg);
    // notification.notifyType === "admin_set" / "mute_user" / ...
    // 显示对应的通知内容
}
```

### 4. 聚合查询：获取用户所有群的通知（新）

**接口**: `1016-33-9` (processorId=1016, jobDispatchId=33, actionId=9)

**请求方式**: POST

**使用场景**: 首页/通知中心，一次返回用户所在所有群的通知，所有通知混在一起按时间倒序排列。

**请求参数**:
```json
{
    "processorId": 1016,
    "jobDispatchId": 33,
    "actionId": 9,
    "token": "当前用户UID",
    "device": 1,
    "doInput": true,
    "newData": {
        "request_uid": "当前用户UID",
        "page": 1,
        "page_size": 20
    }
}
```

**响应格式**:
```json
{
    "success": true,
    "returnValue": {
        "total": 15,
        "page": 1,
        "page_size": 20,
        "notifications": [
            {
                "id": "123",
                "g_id": "group_001",
                "g_name": "技术交流群",
                "notify_type": "admin_set",
                "operator_uid": "user_001",
                "operator_nickname": "张三",
                "target_uid": "user_002",
                "target_nickname": "李四",
                "content": "群主"张三"将"李四"设为管理员",
                "extra_data": null,
                "create_time": "2026-05-15 10:30:00"
            }
        ]
    }
}
```

**响应字段说明**:

| 字段 | 类型 | 说明 |
|------|------|------|
| `total` | int | 用户所有群的通知总数 |
| `page` | int | 当前页码 |
| `page_size` | int | 每页数量 |
| `notifications` | array | 通知列表，所有群混在一起按时间倒序 |
| `notifications[].g_id` | string | 群ID |
| `notifications[].g_name` | string | 群名称 |
| `notifications[].notify_type` | string | 通知类型 |
| `notifications[].operator_uid` | string | 操作者UID |
| `notifications[].operator_nickname` | string | 操作者昵称 |
| `notifications[].target_uid` | string | 目标用户UID（可为null） |
| `notifications[].target_nickname` | string | 目标用户昵称（可为null） |
| `notifications[].content` | string | 通知展示内容 |
| `notifications[].extra_data` | string | 额外数据（JSON） |
| `notifications[].create_time` | string | 创建时间 |

**说明**:
- 返回用户所在所有群的通知，混在一起按 `create_time` 倒序排列
- 每条通知都包含 `g_id` 和 `g_name`，方便前端区分来自哪个群
- 推荐在通知中心/首页使用，一次获取所有通知，无需逐群查询

**前端调用示例**:
```javascript
async function fetchAllGroupNotifications(uid, page = 1, pageSize = 20) {
    const response = await fetch('http://your-server:8081/rainbowchat_pro/rest_post', {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify({
            processorId: 1016,
            jobDispatchId: 33,
            actionId: 9,
            token: uid,
            device: 1,
            doInput: true,
            newData: JSON.stringify({
                request_uid: uid,
                page: page,
                page_size: pageSize
            })
        })
    });
    const result = await response.json();
    if (result.success) {
        return JSON.parse(result.returnValue);
    }
    throw new Error('获取通知失败');
}
```

## 九、测试用例

### MT52 - 入群申请通知测试

1. **用户申请加入需审核的群**
   - 操作：普通用户申请加入设置了审核的群
   - 预期：所有管理员和群主收到 MT52 通知

### MT53 - 入群审核结果通知测试

2. **管理员通过入群申请**
   - 操作：管理员同意用户的入群申请
   - 预期：申请人收到 MT53 通知，`approved: true`

3. **管理员拒绝入群申请**
   - 操作：管理员拒绝用户的入群申请
   - 预期：申请人收到 MT53 通知，`approved: false`

### MT54 - 群管理操作通知测试

4. **设置管理员**
   - 操作：群主设置某成员为管理员
   - 预期：被设置者收到通知，其他管理员也收到通知

5. **取消管理员**
   - 操作：群主取消某成员的管理员身份
   - 预期：被取消者收到通知，其他管理员也收到通知

6. **转让群主**
   - 操作：原群主将群主转让给某成员
   - 预期：**仅新群主收到通知**，内容为"你已成为群主"
   - 预期：原群主**不收到**通知（自己操作不需要通知自己）

7. **解散群聊**
   - 操作：群主解散群聊
