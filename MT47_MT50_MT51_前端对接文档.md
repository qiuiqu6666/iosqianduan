# MT47-MT60 群系统命令 前端对接文档

## 概述

这些协议是群聊系统指令，由服务端主动推送给客户端（S2C），消息类型为 `type = 100`（`TYPE_SYSTEAM_INFO`）。

---

## 本次新增协议

| 协议 | 名称 | 典型触发场景 |
|------|------|--------------|
| MT55 | 群头像修改通知 | 群主或管理员修改群头像 |
| MT56 | 禁言设置变更通知 | 开启/关闭全员禁言、仅群主发言 |
| MT57 | 邀请权限变更通知 | 修改“所有人可邀请 / 仅管理员和群主可邀请” |
| MT58 | 成员隐私保护变更通知 | 开启/关闭成员隐私保护 |
| MT59 | 管理员设置/取消通知 | 设置管理员、取消管理员 |
| MT60 | 入群方式变更通知 | 修改“自由加入 / 需管理员审核” |

---

## 协议基础信息

| 字段 | 说明 |
|------|------|
| **type** | 100（系统消息类型） |
| **from** | "0"（服务端发出） |
| **chatType** | 2（群聊） |
| **protocol** | 对应协议号（见下表） |

---

## MT47 - 通用系统通知

### 触发场景
- 新成员通过邀请加入群聊（通知被邀请者）
- 新成员扫码加入群聊（通知群管理员/群主）

### protocol 值
```
47
```

### 消息体结构 (MsgBody4Group)

```json
{
  "type": 100,
  "from": "0",
  "to": "接收者UID",
  "chatType": 2,
  "protocol": 47,
  "parentFp": "指纹码",
  "content": "通知文案",
  "gid": "群ID",
  "extraData": ""
}
```

### 前端处理逻辑

```javascript
if (msg.protocol === 47 && msg.type === 100) {
    appendSystemMessage({ type: 'common_info', gid: msg.gid, content: msg.content });
}
```

## MT50 - 退群/踢人通知

### 触发场景
- 群成员主动退出群聊（通知其他成员）
- 群主/管理员踢出成员（通知被踢者和其他成员）

### protocol 值
```
50
```

### content 内容规则

| 场景 | content 示例 |
|------|-------------|
| 主动退群 | `"xxx"已退出本群` |
| 被踢出群 | `"xxx"已被"xxx"移出本群` |

### 前端处理逻辑

```javascript
if (msg.protocol === 50 && msg.type === 100) {
    if (content.includes('已退出本群')) {
        refreshGroupMembers(msg.gid);
    } else if (content.includes('移出本群')) {
        refreshGroupMembers(msg.gid);
    }
    appendSystemMessage({ type: 'member_removed', gid: msg.gid, content: msg.content });
}
```


## MT56 - 禁言设置变更通知 🆕

### 触发场景
- 群主或管理员开启/关闭全员禁言（通知所有群成员）

### protocol 值
```
56
```

### extraData 完整结构

```json
{
  "gid": "群ID",
  "operator_uid": "操作者UID",
  "operator_nickname": "操作者昵称",
  "mute_mode": 0,
  "content": ""xxx"解除了群禁言",
  "parent_fp": "指纹码"
}
```

### mute_mode 值说明

| 值 | 含义 | content 示例 |
|----|------|-------------|
| 0 | 正常（解除禁言） | `"xxx"解除了群禁言` |
| 1 | 全员禁言（仅管理员和群主可发） | `"xxx"开启了全员禁言` |
| 2 | 仅群主可发言 | `"xxx"开启了仅群主可发言模式` |

### 前端处理逻辑

```javascript
if (msg.protocol === 56 && msg.type === 100) {
    let extra = JSON.parse(msg.extraData || '{}');
    // 更新群禁言状态
    if (currentChatGid === msg.gid) {
        updateMuteMode(extra.mute_mode);
        if (extra.mute_mode > 0) {
            showMuteNotice();
        }
    }
    appendSystemMessage({ type: 'mute_changed', gid: msg.gid, content: msg.content });
}
```

## MT59 - 管理员设置/取消通知 🆕

### 触发场景
- 群主设置某成员为管理员（通知所有群成员）
- 群主取消某成员的管理员身份（通知所有群成员）

### protocol 值
```
59
```

### extraData 完整结构

```json
{
  "gid": "群ID",
  "operator_uid": "操作者UID（群主）",
  "operator_nickname": "操作者昵称（群主昵称）",
  "target_uid": "目标用户UID",
  "target_nickname": "目标用户昵称",
  "is_set_admin": true,
  "content": ""xxx"设置了"xxx"为管理员",
  "parent_fp": "指纹码"
}
```

### is_set_admin 值说明

| 值 | 含义 | content 示例 |
|----|------|-------------|
| true | 设置为管理员 | `"xxx"设置了"xxx"为管理员` |
| false | 取消管理员 | `"xxx"取消了"xxx"的管理员身份` |

### 前端处理逻辑

```javascript
if (msg.protocol === 59 && msg.type === 100) {
    let extra = JSON.parse(msg.extraData || '{}');
    // 更新群成员角色
    refreshGroupMembers(msg.gid);
    // 如果是被操作者在当前聊天群，显示提示
    if (extra.target_uid === currentUserId) {
        showToast(extra.is_set_admin ? '你已被设为管理员' : '你已被取消管理员身份');
    }
    appendSystemMessage({ type: 'admin_changed', gid: msg.gid, content: msg.content });
}
```

## 统一处理入口

### 消息接收统一处理

```javascript
// 消息接收回调
function onMessageReceived(msg) {
    if (msg.type === 100) {
        switch (msg.protocol) {
            case 47: handleMT47(msg); break;
            case 50: handleMT50(msg); break;
            case 51: handleMT51(msg); break;
            case 55: handleMT55(msg); break;
            case 56: handleMT56(msg); break;
            case 57: handleMT57(msg); break;
            case 58: handleMT58(msg); break;
            case 59: handleMT59(msg); break;
            case 60: handleMT60(msg); break;
        }
        return;
    }
}

// 显示系统消息
function appendSystemMessage(data) {
    let html = `<div class="system-msg ${data.type}">
        <span class="content">${data.content}</span>
        <span class="time">${formatTime(new Date())}</span>
    </div>`;
    $('#chat-messages').append(html);
    scrollToBottom();
}
```

## 注意事项

1. **离线消息处理**：这些系统指令也会存入离线消息表，用户上线后通过离线消息接口拉取
2. **去重**：使用 `parentFp`（指纹码）进行消息去重
3. **聊天记录存储**：`from` 为 "0" 的消息存入聊天记录时显示为"系统通知"
4. **通知范围**：
   - MT47：通知被邀请者
   - MT50：通知被踢者，以及除操作者外的所有群成员
   - MT51：通知除修改者外的所有群成员
   - MT55：通知除修改者外的所有群成员
   - MT56-MT60：通知所有群成员（包括操作者本人也可以收到）
